# Velvet v0.0.8

Reliability fixes for Sonos casting and shared-playlist pages.

## Fixes
- **Sonos — auto-resume when the stream drops.** If a Sonos device goes silent mid-track while it should still be playing (the HTTP stream dropped or the device idled), the player now detects the stopped device and re-casts the current track at the current position automatically — you no longer have to toggle output Web↔Sonos to get audio back. It's guarded so it can't loop and won't interrupt the natural hand-off to the next song at the end of a track.
- **Shared playlist pages play again.** The public share page (`/shared/...`) couldn't load its player under the strict Content-Security-Policy because its logic ran as an inline script. The player code is now an external script and the playlist data is delivered as a non-executable JSON block, so shared links work without relaxing CSP.
