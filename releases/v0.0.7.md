# Velvet v0.0.7

Reliability release focused on server-speaker playback and cross-device resume.

## Fixes
- **Server speaker (MPV cast) — resilient resume.** Playback now recovers on its own after a backend restart, even while the browser tab is in the background. The cast heartbeat runs in a Web Worker (which browsers don't throttle when a tab is hidden), MPV auto-starts on the first cast request after a restart, and an actively-playing stream is no longer stopped by the idle watchdog. A hidden tab no longer pauses the server speaker by mistake.
- **Cross-device queue resume.** Reloading the player on a second device or browser now restores the song that was actually playing or paused — not the first track in the queue. Boot-time restore no longer overwrites the server-side queue before reading it, and the current track is matched by file path so the correct row is selected and scrolled into view.
- **Subsonic — duplicate content hashes.** Song IDs are always disambiguated per file, so Subsonic clients no longer confuse two different files that share a content hash; hash lookups return a stable row.

## Internal
- Content-Security-Policy now sets an explicit `worker-src 'self' blob:` so the cast heartbeat Worker loads correctly.
