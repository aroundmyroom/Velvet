# Velvet v0.1.5 — Sonos casting fixes

Hi-res FLAC plays correctly on Sonos again, and the cast log now tells you which
song is playing.

## Sonos
- **Hi-res FLAC no longer plays silently on Sonos.** When a track was queued via a
  child vpath (e.g. `12-inches`), `/cast-queue` couldn't resolve its `sample_rate`
  — child vpaths aren't indexed separately; their files live under the parent ROOT
  vpath. With the sample rate unknown, hi-res (>48 kHz) detection failed and the
  file was streamed **directly** instead of being transcoded, so Sonos (which can't
  decode 192 kHz) played silence. `/cast-queue` now uses the same child-vpath
  fallback as `/cast`, so hi-res files are transcoded to 48 kHz MP3 and play.
- **The cast log names the song.** The `cast-queue` log line now shows the
  now-playing track and how it's routed, e.g.
  `cast-queue ▶ Alphaville - Sounds Like A Melody [12-inches/A/…flac] (transcoded 192000Hz)`.
  A direct hi-res stream is flagged `(DIRECT … — Sonos may not decode)` so the
  cause is obvious in the logs, and upcoming queued tracks are logged separately.

## Maintenance
- **One-command releases.** `npm run release` (or the `/release` command) now runs
  the whole release sequence — bump, webapp version sync, release-note retention,
  docker pin, tests, commit, push, tag, and GitHub release — with safety guards
  (must be on `main`, blackout-window check, tag-free, changelog + release notes
  required).
