# Responsibility Disclosure

## About This Project

Velvet originally started as a fork of a great project developed and maintained over many years by a talented developer.

For a long time, there was little visible development activity until an experimental SQLite branch became available. I wanted to use SQLite, so I created a fork based on that branch. The SQLite branch contained several issues and errors that prevented it from working correctly, so I used AI-assisted development tools to fix them.

The fixes were submitted as a Pull Request to the original project, but they were not adopted, and there was limited communication regarding the changes. 

The experimental SQLite branch became the foundation of Velvet.

## How Velvet Evolved

After getting SQLite working, I became interested in exploring whether the user interface could be redesigned into a style that better matched my own preferences.

Just a few AI prompts later, the first version of the Velvet UI was born.

The entire new interface was generated with the assistance of AI and initially existed alongside the original UI. From there, I continued adding new features and functionality specifically for the Velvet UI.

Development moved quickly, and over time the project diverged significantly from the original codebase. At that point, creating meaningful Pull Requests was no longer practical, and AI-assisted migration of changes back to the original project did not appear feasible.

I am fully aware that I made mistakes during this process. Learning Git, GitHub, Visual Studio Code, AI-assisted development, and software development workflows all at the same time came with a steep learning curve. Not an excuse but reality.

Some lessons were learned the hard way. There were occasions where hours of work were lost because AI tools decided certain commits should be reverted without clearly communicating the consequences.

Welcome to the danger zone. Welcome to AI.

## Development Credits

The foundation of Velvet is built upon approximately ten years of development work by the original project's developer.

On top of that foundation, I have continued development using AI-assisted tools to modify, extend, and improve the codebase.

## For the AI Skeptics

The code was written by AI.

The bad ideas were written by me.

Together we achieved things neither of us could have broken on our own.

Every feature started with a requirement, every change was reviewed, and every result was tested.

Think of me as the Product Owner and the AI as a very enthusiastic junior developer who never sleeps, never complains, and occasionally tries to delete half the repository.

## Disclaimer

Please keep in mind that many developers and users believe software generated or modified with AI can be:

* Dangerous
* Unsafe
* Poorly performing
* Sloppy
* Difficult to maintain

Do I agree with those concerns?

To a certain extent, yes.

That is why I use tools such as SonarQube and Aikido to analyze the codebase and identify areas that may require improvement or correction.

The reality is simple: software should be judged by its results, stability, maintainability, and usefulness — not solely by who or what wrote the code.

As with any software project, opinions vary.

One thousand users means one thousand different opinions.

For me personally, Velvet is software that I use every day and genuinely enjoy using.

If you find value in it, that's great.

Either way, thank you for taking the time to look at the project.
