# Velvet v0.1.1 — Sonos sleep mode

Adds a direct sleep/wake control for Sonos speakers, using the speaker's
status LED as the visible cue.

## Sonos
- **Sleep mode (opt-in, per admin).** Enable it under **Admin → Sonos**. When on
  and casting to a Sonos device, **pause turns the speaker's status LED off**
  (direct sleep — a paused / zero-volume state, the device stays reachable) and
  **play turns it back on**. Selecting (or re-selecting after a refresh) a Sonos
  device aligns the LED with playback: paused keeps it asleep, playing wakes it.
- **LED control API.** New `GET`/`POST /api/v1/sonos/led` read and set the status
  LED (`SetLEDState`/`GetLEDState`), and `GET`/`POST /api/v1/sonos/sleep` read and
  set the speaker's native sleep timer (`ConfigureSleepTimer`).
- **Admin test panel.** With sleep mode enabled, a test panel exposes
  **Sleep now** / **Wake** with a live countdown plus transport + LED readout, so
  you can confirm the device responds before relying on it in the player. The
  panel is hidden when sleep mode is disabled.
- **Clear guidance in the admin notes.** Sleep mode is a reachable
  paused/zero-volume state — not a power-off. With the device's **Battery Saver**
  enabled it powers off (and becomes unreachable) after ~30 minutes idle; disable
  Battery Saver in the Sonos app for a reachable low-power sleep instead.

## Notes
- Battery Saver itself is a Sonos cloud/app setting and is not exposed on the
  speaker's local API, so Velvet cannot read or change it.
