# v0.0.3 — Single-source versioning

**June 2026**

A small maintenance release that simplifies how Velvet is versioned and cache-busted.

## What changed

- **One file to bump.** The version now lives only in `package.json`. A new sync step (`scripts/sync-webapp-version.cjs`) propagates it to every webapp cache-buster and version string — the player and admin `?v=` query strings, the `VELVET_VERSION` constant, the login version tag, and the shared/mobile entry points.
- **Automatic on boot.** The server stamps the version on every start, so a stale cache-buster can no longer slip through and serve old JS/CSS.
- Run manually any time with `npm run sync-version`.

No user-facing behaviour changes — this is purely about keeping releases consistent and avoiding stale-asset bugs.
