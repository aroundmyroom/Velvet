# Velvet v0.1.0 — Accessibility milestone

This release makes the web player usable with a keyboard and assistive
technology, alongside admin and Sonos improvements.

## Accessibility
- **Keyboard & screen-reader support across the player.** The seek/progress bars
  are now real ARIA sliders (`role="slider"`, keyboard-focusable, current
  position announced as e.g. "1:23 / 4:05"). Icon-only buttons now carry an
  `aria-label` (mirrored from their translated tooltip), and every dialog
  announces itself (`role="dialog"` / `aria-modal`), **traps focus** while open,
  closes on `Esc`, and **restores focus** to whatever opened it.
- **Expanded, discoverable keyboard shortcuts.** New: `/` (jump to search),
  `R` (cycle repeat), and `?` (open an in-app keyboard-shortcuts cheat-sheet) —
  alongside the existing `Space`, `←/→` (seek), `Shift+←/→` (track), `↑/↓`
  (volume), `M` (mute), `S` (shuffle). All documented in **`docs/accessibility.md`**.

## Admin
- **"Backup" is now "Backup & Logs".** The log download and the "write logs to
  disk" toggle were folded into this page (the separate *Logs* page is gone).
  The downloadable bundle is now **only `*.log` files from the last 7 days** —
  small enough to share, with no rotate-audit JSON or stray folders.

## Sonos
- **Offline devices are clearly marked and can't be mis-selected.** In the output
  picker a device goes yellow (checking) → green (ready) → **red (offline)**, and
  an offline device is no longer selectable — no more cast errors from picking a
  device that isn't there.
- **Quieter logs.** Discovery now logs only when a device's reachability
  *changes*, instead of repeating an "unreachable" line every few minutes.

## Other
- Minor login-screen wording tweak.
