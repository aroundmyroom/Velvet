# Velvet v0.1.4 — Sonos Favourites

The "Sonos Radio" view becomes **Sonos Favourites**: it now lists favourites from
every service (Sonos Radio, Spotify, Apple Music, TuneIn, …), plays any of them,
hides the clutter, and shows live now-playing while a favourite is on the speaker.

## Sonos
- **All favourites, not just radio.** `GET /api/v1/sonos/favorites` returns every
  "My Favorites" entry (FV:2), each tagged with its originating `service`.
  Previously only Sonos Radio stations were surfaced, so a Spotify favourite like
  "New Music Friday NL" never appeared — now it does. `radio-favorites` still
  returns the radio-only subset.
- **Play any favourite, any service.** `POST /api/v1/sonos/play-favorite` plays a
  favourite by its FV:2 id, replaying the favourite's own stored `res`/`resMD`
  (which carries the device's service auth token) — so Spotify/Apple Music
  playlists work, not just `x-sonosapi-radio:` stations. Playlist containers are
  enqueued and started from the first track; single streams are set directly on
  the transport.
- **Favourite playback behaves like a real output.** Starting a favourite puts the
  web player into a dedicated Sonos-Favourite mode: the browser is paused and
  muted, the player bar shows the device's **live now-playing** — track title,
  artist, cover and progress, polled every 3 s — and the output button shows the
  room. Pressing **Play**, picking any local track, or choosing **Browser** takes
  control back to the browser and stops the favourite. `transport-status` now
  returns `trackTitle`/`trackArtist`/`trackAlbum`/`trackArt`/`trackUri`.
- **Hide the clutter.** Favourites with no playable resource (the art-less Sonos
  shortcut folders that can't be started via the local API) are concealed by
  default. Admins also get a per-favourite **Hide**/**Show** toggle; both are
  recalled via **Show hidden (N)**. Hide preferences persist in config
  (`sonos.hiddenFavorites`, all users) via `POST /api/v1/sonos/favorite-visibility`.
- **Self-healing default-room IP.** When a cast redirects after DHCP moves the
  speaker, the rediscovered IP is written back to the saved default room (matched
  by UUID), so favourites and casts stop pointing at a dead address.

## Note
The full-screen **Playing Now** view is built around library songs (bio, similar
artists, audio pills) and still reflects Velvet's queue, not an external
favourite — the always-visible player bar is the live indicator for favourites.
