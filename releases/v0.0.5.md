# v0.0.5 — Scanner hardening and cleanup

**June 2026**

A maintenance release focused on scanner resilience, player polish, and repository cleanup.

## Fixes

- **Scanner timeout guard extended to all metadata backfill phases.** Docker scans that hit a single bad FLAC or metadata file during targeted re-parse no longer hang forever. The scanner now applies the same 30-second timeout to art, cue, duration, tech-meta, and album-version refreshes, then records the failure and continues.
- **Timeout logging made actionable.** When a file does time out, the scanner log now includes the exact phase, vpath, scan ID, and file path so the blocking file can be found quickly on the affected system.

## UI

- **Player branding scaled up.** The sidebar logo and centre VU logo were increased in size so the Velvet branding is clearer in the main player.

## Docs

- Clarified in [docs/docker.md](https://github.com/aroundmyroom/Velvet/blob/main/docs/docker.md) that `ghcr.io/aroundmyroom/velvet:latest` can be used directly as well as version-pinned tags.

## Cleanup

- Removed the unused legacy `webapp/alpha` frontend files from the repository.

## Docker

```shell
docker pull ghcr.io/aroundmyroom/velvet:v0.0.5
```

See [docs/docker.md](https://github.com/aroundmyroom/Velvet/blob/main/docs/docker.md) for full setup instructions.
