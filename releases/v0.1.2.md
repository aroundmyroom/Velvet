# Velvet v0.1.2 — Sonos fix

A small fix for Sonos device reachability after a DHCP address change.

## Sonos
- **Fixed: the "Sonos Radio" nav button could disappear after a speaker's IP
  changed.** When a Sonos speaker's DHCP address drifts (e.g. `10.1.1.207` →
  `10.1.1.210`), the saved default-room IP goes stale. The Radio-nav reachability
  probe — and the favourites/browse loaders — were checking that stale address,
  got "unreachable", and hid the menu, even though casting still worked (the cast
  path resolves the live IP). `_sonosTargetIp()` now resolves the default room to
  its **live discovered IP by UUID**, so the nav stays visible and the fix
  self-heals any future DHCP drift.
