# v0.0.2 — Reactive logo & auth fixes

**June 2026**

A polish-and-stability release: the visualizer gets a sound-reactive Velvet logo, and several long-standing playback and auth annoyances are fixed.

## Highlights

- **Sound-reactive logo** — the Milkdrop "Velvet" preset now puts the Velvet logo front and centre, wrapped in effects that move with the music: a hue-cycling halo and logo glow that pulse on bass, rotating aura rays that react to treble, and beat-rings that radiate from the logo.
- **No more 401 spam** — background polls no longer hammer the server with an expired token. Sessions that time out are now cleared cleanly.
- **Clean Sonos hand-off** — switching output to Sonos silences the browser without killing the VU meters or spectrum display.
- **Quieter queue persistence** — a full localStorage no longer floods the log or the database with retries.

## Fixes

- Auth: `api()` clears expired/invalid tokens centrally; pre-login poll guards corrected for `/sonos/devices`, `/radio/schedules/active`, and scan status.
- Sonos: browser output is muted via the cast-mute gain node, leaving the analyser feed intact so meters keep working while casting.
- Queue: localStorage quota overflow retries a smaller window first and throttles the DB-sync fallback.
- Scanner: multi-disc albums may reference a parent-folder cover (`../Folder.jpg`) without a path-safety error.
- Recently Added: sorts by scan-discovery time rather than file modification date.

## Docs & maintenance

- Added a Docker migration guide for users moving from the old `mstream-velvet` image.
- Removed four unused GitHub Actions workflows.
